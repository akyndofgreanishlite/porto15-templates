Localytics, Version 3

When Localytics was founded in 2008, significant technical understanding was required to use app analytics effectively. Today, everyone from project managers to marketers use them to track user behavior across devices and apps.

Localytics asked us to simplify a growing list of features into an intuitive and tunable app, improve upon powerful data analysis capabilities, and integrate messaging features.

Diving into the Details

Designing for mobile first, we simplified the navigation, using icons to represent primary app management, growth, and development tasks. This was the first step in making the app mirror the iterative product cycle of analysis and refinement.

Simple, Yet Powerful

We helped reduce the vast number of controls previously visible on a chart by folding the previous third-level nav items into chart presets and grouping features like chart visual selection into popover menus. This visual simplicity allows customers to quickly change chart views, export data, and find help, and focus on understanding their users.

Charts as Content

I had a breakthough when I realized that the app should treat charts like news articles or image boards, and provide previews that enable users to quickly decide which topics require further examination.

We could help the user focus on the visualizations by reducing cluttered chart axis labels. For charts with time on the x-axis, I achieved this by dictating the label density based on the date range selected.

User Tunable

To improve the chart previews’ relevance to individual users, we’ve allowed them to pin their favorite charts to the parent dashboards.

We also implemented the chart formula in plain English, so any user would be able to understand and change the data sources and splits of a given chart. These formulas are also savable as presets, allowing users to iterate on charts and return to them later.
